Hexo模板和github Pages结合建静态博客

下载git

git是一个版本控制的工具，而github有点类似于远程仓库，用于存放用git管理的各种项目。为了能够在本地使用git管理github上的项目，需要进行一些配置。然后在git bash/cmd（windows）/终端（ubuntu）中运行命令。

申请github账号

申请一个github的账号,然后可以跟着它的教程熟悉一下。如果会的话，当我没说。

检查SSH Keys

首先检查你电脑上现有的ssh key（双系统中两个系统对应的SSH可能会不一样！）

如果提示：No such file or directory 说明你是第一次使用git。那么就执行下一步

生成SSH Key

然后系统会要你输入密码

Enter passphrase (empty for no passphrase):<输入加密串>
Enter same passphrase again:<再次输入加密串>

在回车中会提示你输入一个密码，这个密码会在你提交项目时使用，如果为空的话提交项目时则不用输入。这个设置是防止别人往你的项目里提交内容。
注意：输入密码的时候没有*字样的，你直接输入就可以了。

添加SSH Key到GitHub

在本机设置SSH Key之后，需要添加到GitHub上，以完成SSH链接的设置。

1、打开本地C:\Documents and Settings\Administrator.ssh\id_rsa.pub文件。此文件里面内容为刚才生成人密钥。如果看不到这个文件，你需要设置显示隐藏文件。准确的复制这个文件的内容，才能保证设置的成功。

2、登陆github系统。点击右上角的 Account Settings—>SSH Public keys —> add another public keys

3、把你本地生成的密钥复制到里面（key文本框中）， 点击 add key 就ok了

可以输入下面的命令，看看设置是否成功(git@github.com的部分不要修改)：

如果是下面的反馈：

The authenticity of host ‘github.com (207.97.227.239)’ can’t be established.
RSA key fingerprint is 16:27:ac:a5:76:28:2d:36:63:1b:56:4d:eb:df:a6:48.
Are you sure you want to continue connecting (yes/no)?

没事，输入yes就好!

设置用户信息

现在你已经可以通过SSH链接到GitHub了，还有一些个人信息需要完善的。Git会根据用户的名字和邮箱来记录提交。GitHub也是用这些信息来做权限的处理，输入下面的代码进行个人信息的设置，把名称和邮箱替换成你自己的，名字必须是你的真名，而不是GitHub的昵称。

github上建立仓库

登录后系统，在github首页，点击页面右上角的加号里面的「New Repository」

填写项目信息：
project name：username.github.io(username是你的github用户名)

description： 描述，可不写

注：Github Pages的Repository名字是特定的，后缀一定要有io!点击「Create Repository」 完成创建。

图床

由于github pages只提供300M的空间，写静态博客足矣，但是总的放点图片吧！所以这是你需要一个图床来为你保存图片，所以不好意思七牛云就是这样的云服务!它是免费的在一定的空间内（10G），超过你就得付费了。付费其实这说明了一个好消息你的博客的访问量可以啊，或者坏消息是你上传的图片真的很大了，所以最好是将自己的图片控制在200kb左右吧！
